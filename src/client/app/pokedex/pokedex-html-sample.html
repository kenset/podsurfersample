Angular Templates

Ignore the section and div tags -- they only serve to position the Pokedex on the screen.

The curly braces around pokedexCtrl.pokedex | json interpolate the data located at pokedexCtrl.pokedex, and enable the actual data to show up instead of the literal characters.

json is just an Angular filter that pretty prints the Pokedex.

But how do we actually get at the data? How does Angular know that our Pokedex is located at pokedexCtrl.pokedex?

This is because we're using Controller As syntax.

In some examples you might see people write the following: 

<div ng-controller="PokedexController as pokedexCtrl">
  {{pokedexCtrl.pokedex | json}}
</div>

This is fine, and it definitely works, but it tightly couples the template we use with the controller we use it with. Rather, we prefer to assign the view and controller in the Route file. There, you'll find that we configure the states to use a specific template, with a specific controller, with a specific controller name, denoted with controllerAs.

This, combined with the fact that pokedex is the name of the object we assigned the Pokedex data to in our PokedexController, enables us to reference pokedexCtrl.pokedex in our view.

<section class="mainbar">
  <section class="matter">
    <div class="container">
      <div class="row">
        <div class="col-xs-12">
          <pre>{{pokedexCtrl.pokedex | json}}</pre>
        </div>
      </div>
    </div>
  </section>
</section>
